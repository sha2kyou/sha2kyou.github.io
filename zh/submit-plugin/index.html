插件开发指南

学习如何使用 JavaScript 和我们全面的 API 为 Sidefy 创建强大的插件。

核心函数结构

function fetchEvents(config) {
    // 您的插件逻辑代码
    return events;  // 返回事件数组
}

每个 Sidefy 插件都必须实现 fetchEvents 函数，该函数接受 config 参数并返回 events 数组。

事件对象格式

{
    title: "事件标题",
    startDate: "2024-01-01T10:00:00Z",
    endDate: "2024-01-01T11:00:00Z",
    color: "#3B82F6",
    isAllDay: false,
    isPointInTime: false,
    notes: "可选描述",
    icon: "https://example.com/icon.png",
    href: "https://example.com",
    imageURL: "https://example.com/image.jpg"
}

必需字段:

  • title - 事件标题（字符串）
  • startDate - ISO8601 格式的开始时间
  • endDate - ISO8601 格式的结束时间（除非 isPointInTime 为 true 时必需）
  • color - 十六进制颜色值
  • isAllDay - 是否为全天事件（布尔值）
  • isPointInTime - 是否为时间点事件（布尔值）

可选字段:

  • notes - 事件描述（字符串）
  • icon - 图标 URL（字符串）
  • href - 点击链接（支持 http://、https://、popup://）
  • imageURL - 图片 URL（字符串）

扩展插件文档 - 可用 API

通过插件中的 sidefy 对象访问强大的功能。

HTTP 请求

sidefy.http.get(url)
sidefy.http.post(url, data)

本地存储

sidefy.storage.get(key)
sidefy.storage.set(key, value)
sidefy.storage.clear()

1Mb 限制

日期处理

sidefy.date.parse(dateString)
sidefy.date.format(date, format)

AI 功能

sidefy.ai.chat(prompt)

应用信息

sidefy.app.language()

返回 'en' 或 'zh'

导出和提交流程

按照以下步骤将您的插件提交到社区仓库。

1

开发插件

使用 fetchEvents 函数和可用的 API 创建您的插件。彻底测试以确保兼容性。

在 Sidefy 应用中测试： 使用 Sidefy 内置的插件编辑器直接在应用中编写和测试您的插件代码。在插件管理界面中，您可以编辑代码、运行代码查看事件输出，并调试插件行为以确保与 Sidefy 的兼容性。

2

准备文档

编写清晰的文档，说明插件的功能、要求和使用说明。

从 Sidefy 导出： 使用 Sidefy 的插件导出按钮导出 info.json 和 main.js 文件到以您的插件名命名的目录中。这会创建标准的插件目录结构，准备用于分发和提交到社区仓库。

3

提交到 GitHub

向 SideCalendarPlugins 仓库创建包含插件代码和文档的拉取请求。

//目录名称，同时也是客户端唯一插件id
your-plugin-name/
    ├── main.js # 插件代码
    ├── info.json # 插件基础信息
    └── README.md # 说明文档
访问 SideCalendarPlugins 仓库 →
4

审核和合并

我们的团队将审核您的插件质量、安全性和兼容性，然后将其合并到仓库中。

常见问题

关于插件开发和提交的常见问题。

我可以使用哪些编程语言开发插件？

插件使用 JavaScript 编写，在沙盒环境中运行。您可以使用现代 JavaScript 特性和提供的 sidefy API。

审核过程需要多长时间？

审核过程通常需要 1-3 个工作日。我们会彻底测试每个插件的质量、安全性和兼容性。

提交后可以更新插件吗？

是的，您可以随时通过相同的 GitHub 拉取请求流程更新您的插件。

准备构建您的插件了吗？
