MCP Server: Let AI Assistants Manage Sidefy

FeatureMCPAutomationExperimental
An experimental localhost MCP endpoint that exposes Sidefy plugins, RSS, GitHub, rules, and more as tools for MCP clients.

We’re adding an MCP Server to Sidefy — an experimental feature that exposes a local Model Context Protocol endpoint on your Mac. When enabled, MCP clients (such as Claude Desktop, Cursor, or Claude Code) can read and modify Sidefy configuration while the app is running.

This is the first step toward letting AI assistants help you manage feeds, plugins, GitHub subscriptions, and SideScript rules without clicking through every settings panel.

How It Works

The MCP server runs inside Sidefy and listens on localhost only:

  • Transport: Streamable HTTP (MCP)
  • Endpoint: http://127.0.0.1:<port>/mcp (default port 39281, configurable in Experimental Features settings)
  • Authentication: Authorization: Bearer <token> on every request
  • Requirement: Sidefy must stay running, and experimental features must stay enabled, while an MCP client is connected

To enable it, choose Enable Experimental Features from the Help menu, then open Experimental Features from the menu bar (under ADVANCED) and turn on MCP Server. Set Port if needed (valid range 1024–65535, default 39281), copy the token from the Connection section’s Authorization header example, and regenerate the access token from that panel.

Note: MCP Server is an experimental feature. Settings on the Experimental Features page are reset when experimental features are disabled, and the MCP server stops immediately.

Warning: AI or automated MCP clients may make incorrect changes. Review all actions after they run.

Client Setup

Before adding Sidefy in Cursor, Claude Code, or Claude Desktop:

  1. In the Help menu, turn on Enable Experimental Features and confirm the warning
  2. Open Experimental Features from the menu bar (under ADVANCED — this item appears only after step 1), enable MCP Server, set Port if needed, and confirm the status shows Healthy
  3. Keep Sidefy running and leave experimental features enabled while the MCP client is connected
  4. Copy <token> from the Connection section’s Authorization: Bearer … example; use the endpoint shown there for your client URL — update both if you change Port or regenerate the token

Tool Overview

The server currently exposes 22 tools, grouped by area. Read tools return JSON summaries. Most write tools apply changes immediately and trigger a full event refresh; online plugin config updates use the lighter incremental refresh path instead.

Event Source Plugins

These tools target Sidefy’s top-level event sources (Calendar, RSS, Reminders, GitHub, Tips, Custom Plugins, and so on), not individual custom or online marketplace plugins.

ToolDescription
list_pluginsList event source plugins and their enabled state
set_plugin_enabledEnable or disable an event source plugin by plugin_id (for example com.sidecalendar.rssplugin)

RSS Feeds

ToolDescription
list_rss_feedsList configured RSS feed subscriptions
add_rss_feedAdd a feed by url (optional name, category)
remove_rss_feedRemove a feed by url

GitHub Repositories

ToolDescription
list_github_reposList configured GitHub repository subscriptions
add_github_repoSubscribe to a repository (owner/repo or GitHub URL)
remove_github_repoRemove a repository subscription

Event Rules (SideScript)

ToolDescription
list_rulesList event rules (id, name, enabled state)
get_ruleGet one rule including its full SideScript source
create_ruleCreate a rule from a sidescript body
update_ruleUpdate a rule’s SideScript and/or is_enabled state
set_rule_enabledEnable or disable a rule without editing its body

Rule tools operate on SideScript text. Before creating or editing rules through MCP, read the official syntax guides:

  • SideScript Syntax Guide →
  • SideScript 2.0: Extended TEMPLATE Syntax →

Custom JavaScript Plugins

ToolDescription
list_custom_pluginsList user-defined custom plugins
get_custom_pluginGet one plugin including JavaScript source and config
create_custom_pluginCreate a plugin (name, js_code, optional description, version, config)
update_custom_pluginUpdate an existing plugin’s metadata, code, or config
set_custom_plugin_enabledEnable or disable a custom plugin without changing its source

Custom plugin tools expect JavaScript that defines fetchEvents. See the Submit Plugin page for the plugin API before writing or editing code through MCP.

Online Marketplace Plugins

ToolDescription
list_online_pluginsList downloaded online plugins and their config
get_online_pluginGet one downloaded plugin by UUID, name, or directory name
update_online_plugin_configReplace a plugin’s config and apply it immediately

Config updates for online plugins take effect without restarting Sidefy — the same incremental refresh path used in the plugin editor, including when the editor is already open.

Refresh

ToolDescription
refresh_eventsTrigger a manual refresh of all event sources

Example Workflows

With MCP connected, an assistant can:

  • Add an RSS feed and confirm it appears in list_rss_feeds
  • Subscribe to or remove a GitHub repository subscription
  • Draft a SideScript rule, create it with create_rule, then disable it with set_rule_enabled for testing
  • Update a downloaded marketplace plugin’s API credentials via update_online_plugin_config
  • Enable or disable an event source plugin (such as RSS or GitHub) without opening Plugin Manager

All changes go through the same validation and persistence layers as the native UI.
